Configuration

All configurations are in the avalon.conf.xml file. Configurations have a tree like structure. Avalon is the framework and holds configurations root. Every Block under Avalon has its own configuration inside its block tag. If a Block is made of different component organized in a hierarchy, all these component will have their own configuration node and so on. Each Block provide a specific service to all other Blocks running on top of Avalon. The logger provides a simple logging pattern. Its configurations are under the tag. The same is valid for all other Block including James which has its configuration under tag. Some Block contains in their conf information belonging to other Blocks. As example the logger define a number of channels some for James and its component. Equally the Store contains a definition for the MailRepository which is a class belonging to James. These info comes from the James template file in which is defined which Blocks are needed and wich specific configuration inside those block should be set. So the avalon.conf.xml is the result of merging different template file, one for each server. The file you will find in James distribution is already the result of this merging. We are looking for a smart pattern to write template and merging rules. Comments and suggestions are welcomed.
